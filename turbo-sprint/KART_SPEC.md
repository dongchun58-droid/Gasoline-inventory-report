# TURBO SPRINT — 10초 그랑프리
**카트 레이서 개발 설계서 (Claude Code 실행용)**

- 플레이 환경: iPad Safari + Magic Keyboard (개발·테스트는 Mac 브라우저 병행)
- 스택: Three.js + Vite (vanilla JS, 프레임워크 없음)
- 목표 공수: 8~13시간 (6개 페이즈, 각 페이즈 종료 시 플레이 가능 상태)

---

## 0. 컨셉과 설계 원칙

**한 줄 정의**: 1랩 10~13초짜리 초압축 스프린트 레이스. 드리프트 3단 미니터보, 로켓스타트, 점프 트릭, 아이템, AI 3대 러버밴딩 — 카트 레이서의 손맛을 전부 넣되 한 판이 10초대에 끝난다.

**설계 원칙 3가지 (Claude Code는 모든 판단에서 이 순서를 따를 것)**

1. **60fps > 폴리곤.** 체감 품질의 대부분은 그래픽 해상도가 아니라 속도감 연출(FOV·카메라·파티클·사운드)과 드리프트 손맛에서 나온다. 포토리얼은 목표가 아니다. 목표는 "잘 만든 콘솔풍 스타일라이즈드(셀 셰이딩) 게임"이며, iPad Safari에서 60fps를 절대 사수한다.
2. **외부 에셋 0개.** 3D 모델·텍스처·사운드 파일을 일절 다운로드하지 않는다. 지오메트리는 primitive 조합, 텍스처는 Canvas 2D 생성, 사운드는 Web Audio 합성. 라이선스 이슈가 원천 차단되고, 빌드가 에셋 수급에서 막히지 않는다.
3. **매 페이즈 플레이 가능.** 각 페이즈는 "실행해서 재밌는지 확인 가능한 상태"로 끝난다. 완료 기준 미충족 시 다음 페이즈 진행 금지.

**IP 메모**: 드리프트·미니터보·아이템 같은 게임 메커니즘은 자유롭게 구현하되, 특정 게임사의 캐릭터·명칭·고유 트랙 디자인·로고는 사용하지 않는다. 모든 이름·디자인은 이 문서의 오리지널 명칭을 따른다.

## 1. 기술 스택 & 프로젝트 셋업

```bash
npm create vite@latest turbo-sprint -- --template vanilla
cd turbo-sprint && npm i three
```

- Three.js 최신 안정판. addons는 `three/addons/...` 경로로 import
- 렌더러: `antialias: true`, `outputColorSpace: SRGBColorSpace`, `toneMapping: ACESFilmicToneMapping`(exposure 1.1)
- `renderer.setPixelRatio(Math.min(devicePixelRatio, 1.5))` — iPad 60fps 사수의 핵심
- 포스트프로세싱 사용 시 EffectComposer의 renderTarget에 `{ samples: 4 }`를 줘서 MSAA 유지
- HUD는 WebGL이 아닌 DOM 오버레이(position: fixed)로 — 텍스트가 선명하고 구현이 빠름

### 파일 구조
```
turbo-sprint/
├─ index.html
├─ KART_SPEC.md          # 이 문서
└─ src/
   ├─ main.js            # 씬, 루프(고정 dt 물리 + 가변 렌더), 리사이즈
   ├─ input.js           # 키보드 상태머신
   ├─ track.js           # 스플라인, 도로 메시, 소품, 체크포인트, 부스트패드
   ├─ kart.js            # 차량 물리 + 카트 모델 생성(프로시저럴)
   ├─ ai.js              # AI 컨트롤러 (스플라인 추종 + 러버밴딩)
   ├─ items.js           # 아이템박스, 룰렛, 효과
   ├─ race.js            # 상태머신(타이틀→카운트다운→레이스→결과), 순위 계산
   ├─ camera.js          # 체이스캠, FOV 제어, 스크린셰이크
   ├─ fx.js              # 파티클 풀, 포스트프로세싱(블룸)
   ├─ audio.js           # Web Audio 합성 사운드
   └─ hud.js             # DOM UI
```

물리 업데이트는 고정 타임스텝(1/120s 누적기 방식), 렌더는 rAF. 프레임 드랍 시에도 조작감이 흔들리지 않게 한다.

## 2. 조작 (Magic Keyboard)

| 키 | 동작 |
|---|---|
| ↑ / W | 가속 |
| ↓ / S | 브레이크 · 후진 |
| ← → / A D | 조향 |
| Shift (좌우 모두) | 호핑 → 드리프트 (누르는 동안 유지) / 공중에서 누르면 트릭 |
| Space | 아이템 사용 |
| R | 즉시 리스타트 |
| M | 음소거 토글 |

- `keydown`/`keyup`으로 상태 객체 유지. 화살표·Space는 `e.preventDefault()`로 스크롤 차단
- 키 매핑은 `input.js` 상단 상수 객체 한 곳에 모아 변경 가능하게
- iOS 오디오 정책: 첫 키 입력(또는 탭)에서 `AudioContext.resume()`
- Esc 키는 사용 금지 (iPad Magic Keyboard 모델에 따라 없음)

## 3. 게임 플로우

1. **타이틀**: 로고 + "GRAND PRIX (vs AI 3)" / "TIME ATTACK" 선택 (←→ + Space)
2. **카운트다운**: 3 → 2 → 1 → GO! (각 0.8s 간격)
   - **로켓스타트**: GO 표시 전 0.35초 이내에 가속을 시작하면 1.2s 부스트
   - 그보다 일찍부터 홀드하고 있으면 **휠스핀**: 0.6s간 출발 불가 (연기 파티클)
3. **레이스**: 1랩 스프린트 (설계상 최적 랩 ~10초, 보통 주행 11~13초)
4. **결과**: 순위 + 기록 + BEST(localStorage 저장) + "R = 재도전" — 재도전 루프를 3초 안에 돌게 한다

## 4. 트랙 — "선셋 스카이 서킷"

공중에 떠 있는 네온 서킷. 노을 하늘 위라 스카이박스가 화면의 절반을 차지 → 그라디언트 하늘만 잘 만들어도 비주얼의 절반이 해결된다.

- 전장 약 300m, 도로폭 14m, 공중부양(가장자리 밖 = 낙하)
- `CatmullRomCurve3`(closed) 스플라인 → 곡선을 따라 도로 단면을 압출해 메시 생성
- 제어점 초안 (x, y, z — 튜닝 자유, 전체 구성 유지):

```js
const CONTROL_POINTS = [
  [0, 0, 0],      [55, 0, -4],    [100, 1, -28],  // 홈스트레이트 → 우코너 진입
  [112, 3, -78],  [88, 5, -112],                   // 우헤어핀 (드리프트 구간 1)
  [42, 5, -104],  [18, 4, -72],   [-28, 6, -64],   // S자 (드리프트 구간 2)
  [-72, 10, -30],                                   // 오르막
  [-78, 13, 18],                                    // 점프대 → 대점프
  [-38, 2, 52],   [-8, 0, 24],                      // 착지 → 피니시 직선
];
```

### 트랙 요소 (위치는 스플라인 t값 기준)
| 요소 | 위치 | 스펙 |
|---|---|---|
| 스타트 그리드 | t=0 | 4대 2×2 배치, 피니시라인 체커 텍스처 |
| 아이템박스 1열 | t≈0.18 | 도로 가로로 3개, 간격 4m |
| 부스트패드 | t≈0.30, 0.58, 0.88 | 밟으면 1.0s 부스트, 발광 셰브론(에미시브) |
| 아이템박스 2열 | t≈0.55 | 3개 |
| 점프대 | t≈0.72 | 상향 램프, 이탈 시 수직속도 +9 m/s 부여, 공중 ~0.9s |
| 체크포인트 게이트 | 8등분 (t=0.125 간격) | 보이지 않는 판정면. 순서대로 통과해야 완주 인정(숏컷 방지) + 리스폰 지점 |

- **낙하/리스폰**: 도로 중심선에서 반폭+1m 이상 이탈 & 접지 불가 → 낙하. y가 도로보다 12m 아래로 내려가면 0.8s 페이드 후 마지막 체크포인트에 속도 0으로 리스폰
- **연석**: 도로 양끝 1m는 빨강·흰색 체커 연석(감속 없음, 시각 경고용)

## 5. 차량 물리 (숫자는 이 값으로 시작)

```js
const PHYS = {
  maxSpeed: 28,        // m/s (체감 ~100km/h)
  boostMultiplier: 1.35,
  accel: 18,           // m/s², v가 max에 가까울수록 지수 감쇠
  brake: 30,
  drag: 8,             // 무입력 시 감속
  reverseMax: 8,
  turnRateLow: 2.4,    // rad/s @ 저속
  turnRateHigh: 1.3,   // rad/s @ 최고속 (속도에 따라 선형 보간)
  gravity: 22,
};
```

### 드리프트 & 미니터보 (이 게임의 심장 — 손맛 최우선 튜닝 대상)
- Shift 입력 → **호핑** 0.15s(포물선 +0.4m). 착지 시 좌/우 조향 입력이 있으면 그 방향으로 드리프트 진입(방향 고정)
- 드리프트 중:
  - 카트의 시각적 yaw는 진행 방향 대비 드리프트 방향으로 기본 +18°(슬립 느낌)
  - 실제 선회율: 안쪽 키 홀드 = 1.9 rad/s(조임) / 무입력 = 1.4 / 바깥 키 = 0.9(벌림)
  - 속도는 최고속의 92%로 유지
- **차지 게이지**: 드리프트 지속시간 누적(조임 상태는 1.3배속 충전)
  - 1단 0.7s → 파란 스파크 → 해제 시 **0.5s 부스트**
  - 2단 1.5s → 주황 스파크 → **1.0s 부스트**
  - 3단 2.4s → 핑크 스파크 → **1.5s 부스트**
- 해제 시 헤딩을 진행 방향으로 0.2s에 걸쳐 복귀 + 부스트 발동
- **트릭**: 점프대 등으로 공중에 뜬 상태에서 Shift → 카트 360° 스핀 애니메이션 → 착지 시 0.5s 부스트

### 부스트 공통
- 속도 상한을 `maxSpeed × 1.35`로 해제, 지속시간 갱신형(중첩 시 타이머 리셋)
- 발동 시: FOV 킥 +8, 배기 화염 파티클, 0.15s 스크린셰이크, 부스트 사운드

### 접지·충돌
- 매 프레임 스플라인 최근접점 기반으로 도로면 높이·노멀 산출 → 카트를 노멀에 정렬(경사 대응)
- 카트끼리: 반경 1.1m 구 충돌, 서로 밀어내기 + 상대속도 10% 감속 + "퉁" 사운드
- 스핀아웃(피격·휠스핀): 1.2s간 조작 불능 + 720° 회전 애니메이션, 속도 40%로

## 6. AI (3대)

- 스플라인 추종: 각자 레인 오프셋(-3 / 0 / +3m)을 가진 목표점을 8m 앞에서 룩어헤드 조향
- 목표 속도 = 트랙 곡률 기반(코너 감속) × **러버밴딩 계수**:
  - 플레이어보다 15m 이상 앞 → ×0.92
  - 15m 이상 뒤 → ×1.10
- 드리프트는 물리 생략, 고곡률 구간에서 스파크 파티클 + 슬립 포즈만 연출
- 아이템: 획득 1~2s 후 사용 (터보캔은 직선 구간에서만)
- 라인업(오리지널): **CRIMSON-01**(빨강), **TEAL-02**(청록), **GOLD-03**(금색). 플레이어 = 파랑

## 7. 아이템 (v1은 2종만 — 10초 레이스에선 즉발형만 의미 있음)

- **아이템박스**: 반투명 무지개 회전 큐브(1.2m, 에미시브 + 살짝 굴절 느낌), 획득 시 파열 파티클, 3s 후 리스폰
- 획득 → HUD 슬롯에서 룰렛 0.8s → Space로 사용

| 아이템 | 효과 | 배분 |
|---|---|---|
| 터보 캔 | 즉시 1.2s 부스트 | 1위: 100% / 2~4위: 50% |
| 호밍 로켓 | 바로 앞 순위 카트 추적(선회율 3 rad/s), 명중 시 스핀아웃 1.2s | 2~4위: 50% |

- (스트레치) 오일 슬릭: 뒤에 설치, 밟으면 스핀

## 8. 카메라 & 속도감 연출 (체감 품질의 핵심 모듈)

- **체이스캠**: 카트 뒤 5.5m, 높이 2.4m, 주시점은 카트 앞 4m
  - 위치는 스프링-댐퍼 추적(계수 0.10~0.14) — 코너에서 카트가 화면 안에서 살짝 미끄러지는 느낌
  - 드리프트 중 카메라 롤 ±2° + 드리프트 바깥쪽으로 0.6m 오프셋
- **FOV**: 기본 60 → 속도 비례 최대 72 → 부스트 중 80 (lerp 0.08)
- **스피드라인**: 속도 70% 이상에서 화면 가장자리 방사형 라인 파티클
- **스크린셰이크**: 부스트 발동·피격·착지 시 0.15~0.25s
- 세로 모드 감지 시 "가로로 돌려주세요" 오버레이

## 9. 비주얼 스타일 가이드 (콘솔풍 셀 셰이딩 레시피)

**팔레트** (모든 색은 여기서만 파생):
- 하늘: `#FF9E5E → #FF5E8A → #3D2C8D` 3단 그라디언트 (노을)
- 도로: `#2A2440` (진보라 아스팔트) + 센터라인 `#00E5FF`
- 연석: `#FF3355` / `#FFFFFF` 체커
- 네온 레일·소품: `#FF2E9A`, `#00E5FF`, `#FFD166` (에미시브 1.5~2.5 → 블룸이 먹는 색)
- 카트: 플레이어 `#2E6BFF`, AI는 6장 라인업 색

**머티리얼·라이팅**
- 전 오브젝트 `MeshToonMaterial` + Canvas로 만든 3단 그라디언트맵(어두움 55% / 중간 80% / 밝음 100%) — 셀 셰이딩의 전부
- DirectionalLight(노을색 `#FFB36B`, 강도 2.2) + HemisphereLight(하늘색/바닥색, 0.8)
- 그림자맵 대신 **블롭 섀도**: 카트 아래 원형 그라디언트 스프라이트 — 값싸고 토온 스타일에 더 어울림

**포스트프로세싱**
- `UnrealBloomPass` (threshold 0.85, strength 0.55, radius 0.4) — 네온·스파크·부스트 화염이 여기서 살아남
- 비네트(셰이더패스, 강도 0.25)

**프로시저럴 텍스처 (Canvas 2D → CanvasTexture)**
- 체커(피니시라인·연석), 부스트 셰브론, 하늘 그라디언트, 스파크 스프라이트(방사 그라디언트 원)

**카트 모델 (primitive 조합, ~40폴리곤 감성)**
- 바디: 라운드박스(1.6×0.5×2.2m) + 노즈콘 + 리어윙(박스+기둥)
- 휠: 토러스 4개 — **조향각 반영 + 서스펜션 바운스(사인 진동) + 회전** ← 이 디테일이 퀄리티 체감을 크게 올림
- 드라이버: 캡슐 몸통 + 구 헬멧(팀 컬러, 바이저는 에미시브) — 드리프트 시 몸을 안쪽으로 기울임
- 배기관 2개: 부스트 시 화염 파티클 방출구

**트랙 소품 (InstancedMesh로 반복 배치, 드로콜 절약)**
- 네온 기둥, 공중 부유 링, 응원 풍선, 피니시 아치. 총 드로콜 100 이하 / 삼각형 15만 이하 유지

**파티클 시스템 (`fx.js`, 풀링 최대 500개, Points 또는 InstancedMesh)**
- 드리프트 스파크(단계 색상), 부스트 화염, 착지 먼지, 아이템 파열, 낙하 연기, 결승 컨페티

## 10. 사운드 (Web Audio 100% 합성)

| 사운드 | 합성 방법 |
|---|---|
| 엔진 | saw + square 오실레이터 2개(디튠 8센트), 속도→피치 70~240Hz, LPF 800Hz. 부스트 시 +1옥타브 스윕 |
| 드리프트 | 화이트노이즈 → 밴드패스 1.2kHz, 게인은 드리프트 강도 비례 |
| 미니터보 차지 | 단계 도달마다 짧은 상승 비프(440/660/880Hz) |
| 부스트 발동 | 200→900Hz 0.3s 스윕 + 노이즈 버스트 |
| 아이템 획득 | 3음 아르페지오 (C-E-G, 0.25s) |
| 로켓 발사/명중 | 노이즈 버스트 + 하강 스윕 |
| 카운트다운 | A4 × 3 + GO는 A5 길게 |
| 결승 | 승리 팡파레 4음 (I-IV-V-I 감성) |

- 마스터 게인 하나로 M키 음소거. BGM은 스트레치(130BPM 신스 아르페지오 4마디 루프).

## 11. HUD (DOM 오버레이)

- **상단 중앙: 타이머 `00.00`** — 이 게임의 주인공. 크고 굵게 (system-ui rounded, 두꺼운 외곽선)
- 좌상: 순위 `2nd` (대형, 순위 변동 시 팝 애니메이션)
- 우상: 아이템 슬롯 (룰렛 애니메이션)
- 하단: 트랙 진행 바 — 4개 카트를 색 점으로 표시
- 카운트다운/GO/결과는 풀스크린 오버레이. 결과 화면: 순위, TIME, BEST, "R = RETRY"
- 모든 텍스트는 흰색 + 4px 남색 외곽선(text-shadow 4방향) — 어떤 배경에서도 가독

## 12. 빌드 페이즈 (이 순서대로만 진행)

### Phase 1 — 달리는 재미 (2~3h)
Vite+Three 셋업, 트랙 스플라인·도로 메시, 카트 1대(프로시저럴 모델), 키보드 주행(5장 물리), 체이스캠, 임시 하늘.
**완료 기준: iPad Safari 60fps + "조작만으로 이미 재밌다"는 느낌. 여기서 재미없으면 물리 상수부터 튜닝.**

### Phase 2 — 드리프트 (1~2h)
호핑→드리프트→3단 차지→미니터보, 스파크 파티클, FOV 연출, 부스트패드.
**완료 기준: S자에서 2~3단 차지가 자연스럽게 나오고, 드리프트 랩이 논드리프트보다 명확히 빠름.**

### Phase 3 — 레이스 형식 (1~2h)
상태머신, 카운트다운+로켓스타트+휠스핀, 체크포인트·낙하·리스폰, 타이머, 결승, 결과+BEST(localStorage), R 리스타트.
**완료 기준: 10~13초 완주 루프가 3초 내 재시작으로 무한 반복 가능.**

### Phase 4 — AI & 순위 (1~2h)
AI 3대(6장), 러버밴딩, 실시간 순위 계산(체크포인트+t값), 카트 충돌, HUD 순위·진행바.
**완료 기준: 접전 끝에 2등으로 지면 "한 판 더"가 절로 나옴.**

### Phase 5 — 아이템 (1~2h)
아이템박스, 룰렛, 터보 캔, 호밍 로켓, 피격 스핀아웃, AI 아이템 사용.
**완료 기준: 마지막 직선 역전극이 발생함.**

### Phase 6 — 폴리시 (2~3h)
셀 셰이딩 그라디언트맵 정리, 블룸+비네트, 노을 스카이+소품 배치, 전체 사운드, 점프 트릭, 컨페티, 타이틀 화면, HUD 애니메이션.
**완료 기준: 플레이 영상을 찍어서 누군가에게 보내고 싶어짐.**

## 13. 성능 예산 (iPad Safari 기준, 초과 시 폴리시보다 최적화 우선)

- 60fps 고정 (ProMotion 120은 무시, 60 캡)
- 드로콜 < 100 (소품 InstancedMesh 필수), 삼각형 < 150k
- 파티클 풀 ≤ 500, 그림자맵 사용 금지(블롭 섀도), pixelRatio ≤ 1.5
- 매 프레임 GC 유발 금지: 벡터·배열 재사용(임시 Vector3는 모듈 스코프에 미리 생성)

## 14. 실행 & iPad 배포

```bash
npm run dev -- --host
```
→ Mac과 같은 Wi-Fi의 iPad Safari에서 `http://<맥의IP>:5173` 접속. Magic Keyboard로 즉시 플레이.

```bash
npm run build   # dist/ → EC2 nginx 정적 서빙 또는 GitHub Pages
```

```html
<meta name="viewport" content="width=device-width, initial-scale=1, viewport-fit=cover, user-scalable=no">

```
